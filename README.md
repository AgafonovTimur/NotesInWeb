

###  Midi в web странице

Однофайловое HTML-приложение для отображения на весь экран названия ноты/аккорда, взятых на MIDI-клавиатуре, в реальном времени. Предназначение - визуальный помощник при импровизации, разборе аккордов или обучении.

для передачи в DAW нужен loopbe или аналог

## Возможности

- **Определение аккордов в реальном времени** через Web MIDI API — слушает все подключённые MIDI-входы одновременно.
- **Распознавание широкого набора аккордов**: одиночные ноты, трезвучия (maj, min, dim, aug, sus2, sus4, "5"), септаккорды (7, maj7, m7, m7b5, dim7, dimM7, minmaj7, augM7, 7sus4, 7b5, 7#5), секстаккорды (6, min6, 6add9, add9, m add9), нонаккорды (9, maj9, m9, minmaj9) и одиннадцатые (11, maj11, m11).
- **Пасстру MIDI**: пересылает входящие сообщения на выбранный MIDI-выход (например, loopMIDI), с защитой от эха и дублей сообщений между несколькими входами.
- **Окно захвата аккорда**: ноты, нажатые почти одновременно, группируются в один аккорд, а не мигают по одной.
- **Номер обращения аккорда** — отдельная строка, показывает цифру обращения (1, 2, 3…), если в басу не тоника аккорда.
- **Ступень лада** — отдельная строка, показывает римскую цифру ступени (I, ii, V+ и т.д.) относительно выбранной тоники/лада, с учётом регистра (заглавная/строчная) и символа (+/°) по качеству аккорда.
- **Выбор тоники и лада** (мажор, натуральный/гармонический/мелодический минор, дорийский, фригийский, лидийский, миксолидийский, локрийский) — нужен только для отображения ступени лада.
- **Индивидуальный выбор диез/бемоль** для каждой из 5 "чёрных" нот (C#/Db, D#/Eb, F#/Gb, G#/Ab, A#/Bb).
- **Масштабирование** (зум) отдельно для: основной строки ноты/аккорда, строки обращения, строки ступени лада.
- **Сохранение настроек** в `localStorage` — все переключатели, тоника/лад, зум восстанавливаются при перезагрузке страницы.
- Скрываемая панель настроек (MIDI вход/выход, зум, диезы/бемоли, тоника/лад, чекбоксы отображения).

## Горячие клавиши

| Клавиша | Действие |
|---|---|
| `Y` / `Н` | Показать/скрыть панель настроек |
| `+` / `=` | Увеличить масштаб основной строки (нота/аккорд) |
| `-` | Уменьшить масштаб основной строки |
| `Shift` + `-` | Сбросить масштаб основной строки до 100% |
| `B` / `И` | Переключить диез↔бемоль сразу для всех 5 нот |
| `9` | Уменьшить размер строки "номер обращения" |
| `0` | Увеличить размер строки "номер обращения" |
| `7` | Уменьшить размер строки "ступень лада" |
| `8` | Увеличить размер строки "ступень лада" |
|useFlats = true | отображение диезов или бемолей( так же чек боксв настройках сразу отмечен checked)|

Также в панели настроек есть кнопки `+`/`-` для увеличения/уменьшения нот (кнопка `Shift` + `-` для сброса на 100%).

## Что можно поменять в самом файле (без знания программирования)

Все настраиваемые значения вынесены в блок `:root { ... }` в начале `<style>`:

| CSS-переменная | За что отвечает |
|---|---|
| `--bg-color` | Цвет фона страницы |
| `--text-color` | Цвет текста основной строки (нота/аккорд) |
| `--font-size-base` | Базовый размер шрифта основной строки (при зуме 100%) |
| `--panel-top` | Положение панели настроек по вертикали |
| `--inv-color` | Цвет текста строки "номер обращения" |
| `--inv-font-size-base` | Базовый размер шрифта строки обращения (при зуме 100%) |
| `--inv-top`, `--inv-left` | Позиция строки обращения (отступы сверху/слева) |
| `--degree-color` | Цвет текста строки "ступень лада" |
| `--degree-font-size-base` | Базовый размер шрифта строки ступени лада (при зуме 100%) |
| `--degree-top`, `--degree-left` | Позиция строки ступени лада (отступы сверху/слева) |

Остальные переменные (`--zoom`, `--inv-zoom`, `--degree-zoom`) — служебные, изменяются автоматически горячими клавишами/кнопками, их вручную трогать не нужно.

## Что можно поменять в коде (JS-константы)

- **`CHORD_CAPTURE_WINDOW_MS`** (по умолчанию 50 мс) — длительность окна захвата аккорда: сколько ждать без новых нажатий, прежде чем отобразить аккорд.
- **`ECHO_WINDOW_MS`** (по умолчанию 400 мс) — окно защиты от MIDI-эха (когда DAW с MIDI Thru отправляет обратно то, что мы сами отправили на выход).
- **`DUPLICATE_WINDOW_MS`** (по умолчанию 30 мс) — окно защиты от дублирующихся сообщений при прослушивании нескольких MIDI-входов одновременно.
- **`NOTE_OFF_DELAY_MS`** (по умолчанию 60 мс, объявлена, но фактически логика Note Off отключена — см. ниже) — защита от дребезга.
- **`CHORD_TEMPLATES`** — список распознаваемых типов аккордов с их интервалами от корня; можно добавлять свои типы аккордов.
- **`QUALITY_MAP`** — соответствие типа аккорда регистру и символу римской цифры ступени (upper/lower, +/°).
- **`MODE_INTERVALS`** — интервалы ступеней для каждого из 9 ладов.

## Особенности поведения

- **Ноты не гаснут по отпусканию клавиши** — текущая реализация игнорирует Note Off/Note On с velocity 0 (`return` в обработчике). Отображаемый аккорд/нота сохраняются на экране, пока не будет взят новый аккорд. Это сделано намеренно (например, для съёмки, где нужно, чтобы аккорд не пропадал сразу после снятия руки с клавиш).
- Если сочетание нот не соответствует ни одному известному шаблону аккорда, отображается просто список названий нот через пробел, а строки обращения/ступени лада для такого случая не показываются.
- Для одиночной ноты строки обращения и ступени лада: обращение не показывается, ступень лада (если тоника/лад выбраны) показывается с "мажорным" качеством по умолчанию.
- Требуется браузер с поддержкой Web MIDI API (Chrome/Edge). При открытии файла напрямую (`file://`) доступ к MIDI может быть заблокирован браузером — нужно открывать через локальный сервер (`http://localhost`).

2. Разрешите доступ к MIDI-устройствам.
3. Нажмите `Y`, чтобы открыть панель настроек и выбрать нужный MIDI-вход/выход, тонику/лад, диезы/бемоли.
